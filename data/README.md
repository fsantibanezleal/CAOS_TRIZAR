# data/ — the data contract + layout

Governed by the **two data contracts** of ADR-0057, instantiated for ChancaDEM's crusher-comminution domain.

## Layout

| Path | What | Git |
|---|---|---|
| `examples/operating.csv` | a tiny sample of operating points that PASS Contract 1 (clone-verify) | committed |
| `derived/models/*.onnx` | the surrogate MLP + denoising AE, exported for live in-browser inference | committed (small) |
| `derived/{scaler,ae_scaler,ae_threshold}.json` | the frozen inference contract (z-scoring + AE p99) | committed |
| `derived/surrogate_metrics.json` | held-out per-output R²/MAPE + the P80-monotone-vs-CSS gate | committed |
| `derived/case-results.json` | the 17 cases' engine outputs (baked by the SAME TS engine the browser runs) | committed |
| `derived/<case>/trace.json` | the compact per-case replay artifact (Contract 2) | committed (regenerated by the pipeline) |
| `derived/manifests/` | per-case `<case>.json` (Contract 2) + the flat `index.json` | committed |

The committed `derived/` artifacts ARE the offline lane's real outputs — the DEFAULT pipeline (`python -m
chancalab.pipeline all`, numpy-only) rebuilds the per-case traces + manifests from `case-results.json` +
`surrogate_metrics.json`, so a clone replays without torch or Node. `pipeline all --retrain` regenerates everything
(Node sweep of the TS engine → torch train → ONNX; then re-bake `case-results.json`).

## CONTRACT 1 — ingestion (operating point + feed PSD → pipeline) — the *bring-your-own-operating-point* gate

Defined in `data-pipeline/chancalab/io/contract.py`. An operating row is **accepted** iff it satisfies the schema;
physically-meaningless rows are **rejected** with a reason; out-of-machine-envelope-but-plausible rows are
**flagged** (the surrogate is extrapolating and the deep-AE anomaly score is the live guard).

| Column | Unit | Rule |
|---|---|---|
| `machine` | — | ∈ {cone-sec, cone-tert, cone-short-head, gyratory, jaw}; else **reject** |
| `cssMm` | mm | > 0; `cssMm > 2.5× feedX63Mm` → **reject** (CSS wider than the feed top); `cssMm ≥ feedX63Mm` → **flag** (pass-through) |
| `throwMm`, `speedRpm`, `feedX63Mm`, `feedM`, `oreAxb` | mm / rpm / mm / – / – | numeric, positive; **flag** if outside the machine's real envelope |

A `validate_psd(edges, passing)` guard checks a bring-your-own feed PSD (descending sieve edges, monotone passing).

## CONTRACT 2 — artifact (pipeline → web)

Each run writes a compact per-case trace (`derived/<case>/trace.json`, schema `chancadem.trace/v1`) + a manifest
(`derived/manifests/<case>.json`, schema `chancadem.manifest/v2`) recording the category/stage, seed, engine+version,
the shared ONNX, the trace byte size, the lane/gate verdict, the Contract-1 flags, and the case metrics.
`frontend/src/lib/contract.types.ts` mirrors these so any drift fails the web build; `scripts/check_artifacts.py`
(CI) enforces manifest⇄trace byte-consistency.

## Provenance / honesty

The physics engine (Whiten 1972 population balance + Evertsson 2000 capacity + Bond/JKMRC breakage) is the source of
truth, calibrated to published cone data (Duarte et al. 2021, DOI 10.3390/min11111256); the surrogate emulates IT,
NOT a real plant. The CP1/CI1 controls are deliberate negative/invalid cases (the engine must show ~zero reduction
or flag invalidity). No real plant data is used or claimed.
